# 产品需求文档 (PRD)：党委督办管理系统

## 1. 产品概述
“党委督办管理系统”是为医院（或机构）管理层及督查部门量身打造的重点任务督办系统。旨在将党委决策、院长办公会决议及年度工作重点转化为可追踪、可量化、可闭环的明确任务，并提供从宏观看板到微观明细追踪的全流程管理闭环，助力管理效能提升及战略目标落地。

---

## 2. 功能模块与详细需求参数

### 2.1 模块一：全景看板 (Supervision Dashboard)
**功能定位**：为院内管理层提供全局视角的实况监控，通过数据分析与多维图表辅助宏观决策。

#### 2.1.1 顶部数据过滤组件
*   **年度切换**：下拉列表，按年度筛选（例如：2024年度、2025年度、2026年度）。
*   **部门选择**：支持选定“全院所有部门”或指定具体科室（如：党办、院办、医务处等），切换后看板所有图表联动刷新。

#### 2.1.2 核心业务看板 (KPI Cards)
*   **项目总数**：当前筛选条件下的任务项目总和。
*   **进行中**：当前状态为主线状态“进行中”的任务数量。
*   **已完成**：当前状态为“已完成”的任务数量。
*   **逾期预警**：存在滞后风险或已经逾期的关键任务数量（标红高亮展示）。

#### 2.1.3 可视化分析图表
*   **大项分类分布 (雷达图)**：
    *   **参数**：展现重点任务在不同职能大类（党的建设、医疗质量、运营管理、学科建设、人才培养、科研创新等）的分布情况（按满分/最高计数值雷达辐射展示）。
*   **各部门项目执行情况 (堆叠柱状图)**：
    *   **参数**：展示医院各科室作为“牵头部门”时，其肩负任务的推进健康度；包含已完成（绿）、进行中（蓝）、逾期（红）的堆积柱状呈现。

#### 2.1.4 督办预警动态看板 (Alerts List)
*   **参数**：滚动展示存在执行风险的任务状态。
    *   **字段**：任务名称、责任部门、动态状态标签（即将逾期、进度滞后）、预警级别（warning预警/danger严重滞后）、触发时间（如2小时前）。

---

### 2.2 模块二：任务清单管理 (Task List Library)
**功能定位**：督办任务的台账管理，支持督查室干事与系统管理员对具体任务进行增删查改和状态追踪。

#### 2.2.1 综合检索过滤区
*   **状态选项卡**：水平铺开的快捷状态过滤（全部、未开始、进行中、已完成、滞后）。
*   **年份过滤**：精准指定待筛选年度。
*   **关键字搜索**：支持对“任务目标”或“责任部门”模糊查询。
*   **分类多选**：支持对不同的大类栏目（如党建、医疗质量）进行交叉勾选组合查询。
*   **责任领导**：支持点选具体院领导（如张书记、李院长），查询其分管挂靠的具体任务台账。

#### 2.2.2 任务清单列表 (Data Table)
*   **展示字段**：
    1.  **项目编号** (ID/序号)
    2.  **分类** (Category)
    3.  **重点任务/目标** (Target，主展示内容)
    4.  **院领导** (Leader)
    5.  **牵头部门** (Department)
    6.  **协同部门** (Co-Department)
    7.  **任务状态** (Status：已完成、进行中、滞后、未开始，配以红绿蓝特色Badge)
*   **操作**：详情跳转、编辑任务、删除任务。
*   **支持外部数据导入**：支持Excel/CSV附件模板一键导入“导入任务”。

#### 2.2.3 新建/编辑任务表单模块 (Modal Context)
*   **所属年度**：单选（下拉框选择，自动带出当前筛选年份）。
*   **栏目分类**：不仅支持选择已有预设的大项如“后勤保障、科研创新”，还必须支持“**+ 新建分类**”，允许系统管理员自行扩展动态分类（自定义输入）。
*   **重点任务/目标**：文本区域（Textarea），详细描述该任务的最终KPI与目标要求。
*   **院领导责任人**：下拉选择指定的院级分管领导。
*   **牵头部门**：科室层级书（Tree-select）选取，明确主体责任归属。包含两大一级支柱“行政职能部门”与“临床业务部门”。
*   **协同部门**：支持选中多个部门或标识“无”。
*   **当前整体状态**：初始指定状态评估（未开始、进行中等）。

### 2.3 模块三：督办详情及跨层级流转闭环 (Supervision Detail & Workflow)
**功能定位**：全方位透视单一任务进程。连接一线执行部门的“填报”、督导部门的“复核”及院领导的“批示”。

#### 2.3.1 任务基本信息区块
*   展示该任务的全要素基础信息，同时顶部配以直观的**整体完成进度条** (Progress Bar %展示)。

#### 2.3.2 阶段性任务拆解 (Phased Subtasks)
*   将年度长周期任务解构为分期目标表单（以卡片/列表形式并排展示）。
*   **阶段信息字段**：
    *   **阶段名称**：如“第一季度”、“启动期”、“专项整治阶段”。
    *   **时间节点** (Time Node)：明确交付/截止日期。
    *   **阶段任务内容** (Content)：具体的执行动作安排描述。
    *   **考核指标 (KPI)**：作为对该阶段验收的量化标准卡尺。
    *   **阶段办理状态**：系统级硬性状态流转，如：未开始 -> 已填报 -> 待审核 -> 已完成 / 驳回。

#### 2.3.3 填报与进度追踪控制栈 (Action Area)
*   **进展填报上报 (执行层视角)**：
    *   必须选择具体针对“哪一个子阶段的拆解任务”或“总体任务”。
    *   可录入具体落实进展文本汇报。
    *   支持附件上传（多文件，台账资料留痕证明，如签到表PDF、现场图片等）。
*   **领导查看与批示反馈 (管理层视角)**：
    *   院领导针对提交上来的台账和内容进行专项批复和工作指导。

#### 2.3.4 全周期督办记录纪实 (Traces / Timeline)
*   **时间轴面板**：串联该任务的生命周期。
*   **履历属性**：
    *   记录动作发生时间、操作执行人及对应身份。
    *   明确记录该日志的“操作类型”（批示、填报、系统任务下达、审核确认）。
    *   展示日志正文、所伴随状态的变异情况，及可直接点击下载和预览的历史附件存档资料。

## 3. 业务管理闭环逻辑
1. **决策下发**：系统管理员/督办部门通过`任务清单 -> 新增任务`派发分解年度重大事项。
2. **计划拆解**：在任务详情中梳理制定出按季度或周期的`次级/阶段性目标`。
3. **一线执行**：临床与行政科室干事定期通过该平台对对应阶段任务提交`进展填报（含举证附件）`。
4. **院内督导**：督导室检核台账并进行系统状态流转（例如从未开始变更为待审核/已完成）。
5. **高层批示**：院领导实时在`全景看板`获知整体健康度和偏离风险情况，对重点事项发送`批示指导`，直接反馈至督查纪实记录内产生闭环。
