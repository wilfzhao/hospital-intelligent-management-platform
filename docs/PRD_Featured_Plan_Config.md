# 产品需求文档 (PRD)：特色考评方案配置模块

## 1. 模块概述
本模块隶属于“方案管理 > 特色考评方案”，主要承载**分类考核规则的制定**。
针对具有明显业务差异的单元（如：内科系、外科系、医技、职能部门），系统支持运营管理者在一个考评方案下，按照不同“类型”分别配置其**适用范围**及**评价体系（维度与指标规则）**，以实现“分类施策、精准评价”的管理目标。

---

## 2. 页面整体布局与导航
页面采用主流的**分侧工作台**设计，可划分为三大块：

1. **顶部操作栏**：
   - 居左展示“返回”按钮与当前正配置的方案名称（例如：`2024年度学科综合考评方案 - 配置向导`）。
   - 点击“返回”即安全退出配置向导，返回上一级的方案列表。
2. **左侧管理栏（考评类型模板切换区）**：
   - 竖排罗列出所有在此方案下属的“考评类型”（如：默认类型、内科、外科、医技、职能等）。
   - **点击交互**：点击任意一项类型分类（选中态高亮），右侧的主体工作区将自动刷新为您当前所选类型（如“外科类型”）的数据配置视图。
3. **右侧主配置工作区**：
   - 本区域顶部包含两个并列的选项卡（Tab）：**「考核范围配置」**与**「考核维度配置」**。点击即可在人员划分与指标搭建两种工作模式间无缝切换。

---

## 3. 选项卡 1：考核范围配置
**核心场景**：解决“当前所选考评类型，要考核哪些具体的科室或个人？”的问题。

页面呈左右双栏穿梭框形态：
- **左半区（考核对象选取区）**：
  - **搜索框**：支持直接输入科室名称或人员姓名进行模糊模糊匹配。
  - **全院结构树**：展示医院组织架构及人员列表。每一项左侧配有复选框。
  - **级联操作逻辑**：支持逐个勾选项，同时支持“父子节点级联选择”（例：只要勾选了“内科学”这一父节点，其下属的消化内、心内等子科室都会被一键自动全选）。
- **右半区（已选清单管理区）**：
  - 以独立的标签卡片（Tag）形式罗列出所有生效的“被考核对象”。
  - **移出操作**：鼠标悬停任意标签，右上角显示“X”关闭按钮，点击可将该科室/个人移除出此类型的评定范围。
  - **批量清空操作**：右上角配备“清空全部”快速按钮，用于重置该模板的选择范围。

---

## 4. 选项卡 2：考核维度配置（本次改版重点核心）
**核心场景**：解决“当前类型具体要考核哪些总能力、哪些明细考核指标、各占方案里多少分、怎么折算分数？”的问题。

配置面板同样采用分列布局：“维度体系”在左，“底层指标设置”在右。配合顶部的全局总计分提示（如“方案总分1000分”）。

### 4.1 测评体系构建区（左侧板块）
- **层级展示**：树状纵列展示一、二、三级考核维度（如：`医疗能力 -> 手术质量 -> 四级手术占比`）。
- **分数直观展现**：每一个节点均会显式标注它的**权重（%）**和**最终绝对分值**。
- **维度增删改**：
  - 支持新增根维度，亦支持悬浮在现有维度点击「添加子维度」。
  - 【正在完善中的潜在逻辑支持维度的拖拽或名称的双击重命名。】
- **联动触发（关键操作点）**：用户的点击选择必须下钻。当用户且仅当用户**点击了最末端（不含子级）的“叶子维度节点”**（系统会有蓝色高亮提醒），右侧板块才会出现当前该维度下的最终指标列表绑定与参数配置面板。未选中前右侧则展示占位空图。

### 4.2 指标配置与计分规则区（右侧板块）
这是近期变动的核心业务区域，用于完成“特定业务指标与给分方法的规则设定”。

- **关联新指标**：右上角的添加入口。点击后将弹出系统“指标库精选弹窗”，支持在整个医院指标库大盘中跨维度搜索指定指标，勾选确认后即追加到下方的配置表格内。
- **明细配置数据表**：
  以清单表格直列展示针对该维度挂靠的所有底层指标数据，每一行支持快速直接（Inline）输入更改，内容包含：
  
  1. **指标名称**：当前所绑定的关键指标展示项（例如：临床路径规范执行率）。
  2. **计分方法（*近期核心更新功能*）**：提供直观的下拉菜单切换。目前强制支持两种评价模型，并配以蓝紫两种色彩底纹来让用户迅速区分计分逻辑：
     - **达成率计分法**：按照常规实际完成数占比直接转换为最终得分（默认选项）。
     - **区间赋分法**：为满足复杂的非线性考核准则。当选择此项时，意味着系统后台将运用区间配置规则（例如，处理类似于“指标结果≥90%计满分或最高给15分；指标≥80%给10分；未开展该特定业务予以强制性豁免赋满分或0分”及存在硬性扣分底线的复杂分段阶梯计分场景逻辑）。
  3. **指标权重 (%)**：数字输入框，指定该指标在其维度层内部占比（如填入 40%）。
  4. **指标分值**：数字输入框，指定该指标在总分中的最终核算配分上限。
  5. **操作**：行末尾提供垃圾桶图标，点击可解除当前指标与维度的挂靠关系。

## 本次迭代价值总结
本次通过对考核范围选取和指标计分模型（特别是扩展对于区间赋分规则的方法落位与声明），使得业务团队无需将所有评判条件（如豁免规则、特殊场景满分规则）一股脑塞进复杂的判定条件引擎中，而是通过在方案维度下直接声明特定的“区间级计分计算口径”，从交互链路上极大地释放了前端配置的灵活度与后端数据核算的兼容性差异。
